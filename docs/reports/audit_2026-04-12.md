## 🔴 Critical Issues (Phải sửa ngay)
1. Cấu hình CORS quá mở (Allow All Origins)
   - File: `backend/main.py` (Dòng 19)
   - Nguy hiểm: Hiện tại API đang cấu hình `allow_origins=["http://localhost:5173", "*"]`. Dấu `*` đi kèm với `allow_credentials=True` sẽ khiến cho mọi trang web trên Internet đều có thể gửi request có chứa cookie/credentials ngầm tới API của hệ thống. Hacker dựng một trang web độc hại có thể trộm dữ liệu dễ dàng.
   - Cách sửa: Xoá bỏ `"*"` trong danh sách `allow_origins`.

2. Lỗi băm (Hash) địa chỉ IP không ổn định và không an toàn
   - File: `backend/main.py` (Dòng 52)
   - Nguy hiểm: Việc dùng hàm `hash(client_ip)` mặc định của Python rất nguy hiểm và chạy sai. Trong Python, giá trị của hàm `hash()` sẽ bị thay đổi mỗi khi khởi động lại server (mỗi process có một seed ngẫu nhiên). Hơn nữa, nó không phải là mã hoá bảo mật một chiều thực sự.
   - Cách sửa: Nên dùng thư viện `hashlib` cụ thể là `hashlib.sha256()` để mã hóa IP thay vì dùng hàm `hash()` mặc định.

1. Hardcode mật khẩu cơ sở dữ liệu làm cấu hình dự phòng
   - File: `backend/database.py` (Dòng 8)
   - Vấn đề: Password `devops_password` đang nằm lồ lộ trong chuỗi fallback `os.getenv("DATABASE_URL", "...")`. Nếu lỡ push file này lên Git public (như GitHub), toàn bộ thế giới sẽ biết mật khẩu mặc định của database của anh.
   - Cách sửa: Không nên để mật khẩu trong code. Giữ mọi thông tin nhạy cảm ở trong file `.env` và đảm bảo `.env` đã được liệt kê trong `.gitignore`.
