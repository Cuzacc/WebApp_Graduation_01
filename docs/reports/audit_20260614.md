## 🔴 Critical Issues (Phải sửa ngay)

### 1. Lỗ hổng đọc tệp tin tùy tiện (Path Traversal) trong API tải ảnh
- **File:** [backend/main.py](file:///f:/DevOps/web/graduation_01/backend/main.py#L194-L211)
- **Nguy hiểm:** Hacker có thể điền đường dẫn dạng `/uploads/../../../../etc/passwd` hoặc `.env` để vượt qua thư mục chỉ định và tải bất kỳ tệp tin hệ thống nào chứa mật khẩu cơ sở dữ liệu, API keys hoặc mã nguồn.
- **Cách sửa:** Chuẩn hóa đường dẫn bằng cách kiểm tra thư mục gốc tuyệt đối hoặc giới hạn chặt chẽ tên tệp bằng `os.path.basename`.
  ```python
  # Code đề xuất:
  local_filename = os.path.basename(path_parts[1])
  local_path = os.path.abspath(os.path.join("uploads", local_filename))
  if not local_path.startswith(os.path.abspath("uploads")):
      raise HTTPException(status_code=400, detail="Đường dẫn không hợp lệ")
  ```

### 2. Lỗ hổng giả mạo yêu cầu từ phía máy chủ (SSRF) trong API tải ảnh
- **File:** [backend/main.py](file:///f:/DevOps/web/graduation_01/backend/main.py#L213-L233)
- **Nguy hiểm:** Hacker có thể gửi tham số `url` trỏ về các địa chỉ IP nội bộ như `http://127.0.0.1:8000/api/admin/stats`, địa chỉ các container trong Docker hoặc GCP Metadata Server `http://169.254.169.254/` để đánh cắp token tài khoản dịch vụ đám mây.
- **Cách sửa:** Kiểm tra tên miền của URL và chỉ cho phép tải xuống nếu nó trỏ đến các nguồn được phê duyệt trước (như Google Cloud Storage `storage.googleapis.com` hoặc local).
  ```python
  # Code đề xuất:
  parsed_url = urlparse(url)
  if parsed_url.netloc and not parsed_url.netloc.endswith("storage.googleapis.com"):
      raise HTTPException(status_code=400, detail="Không hỗ trợ tải tệp từ nguồn này")
  ```

### 3. Rò rỉ kết nối cơ sở dữ liệu (Database Connection Leak) trong Middleware ghi log
- **File:** [backend/main.py](file:///f:/DevOps/web/graduation_01/backend/main.py#L87-L105)
- **Nguy hiểm:** Trong khối mã `try-except` ghi log traffic của middleware, tệp `db.close()` nằm ở cuối khối `try`. Nếu có bất kỳ lỗi nào xảy ra trong quá trình ghi log (ví dụ: DB khóa hoặc quá tải), lệnh `db.close()` sẽ bị bỏ qua. Kết nối DB bị treo và rò rỉ, dần dần làm cạn kiệt pool kết nối và gây sập cơ sở dữ liệu ("Too many clients").
- **Cách sửa:** Đặt lệnh đóng kết nối DB trong khối `finally` để đảm bảo kết nối luôn luôn được đóng bất kể ghi log thành công hay thất bại.
  ```python
  db = None
  try:
      # ...
  finally:
      if db:
          db.close()
  ```

- **File:** [nginx/nginx.conf](file:///f:/DevOps/web/graduation_01/nginx/nginx.conf)
- **Nguy hiểm:** API gửi lời chúc `/api/wishes` cho phép đính kèm ảnh & video dung lượng lớn (lên tới 15MB - 50MB) nhưng không được giới hạn tần suất gửi. Một người dùng xấu có thể viết script tự động gửi hàng nghìn file để làm tràn đĩa cứng máy chủ (Disk Full DoS) hoặc tiêu tốn chi phí GCP Storage.
- **Cách sửa:** Cấu hình `limit_req` trong Nginx cho endpoint `/api/wishes` tương tự như API login (giới hạn tối đa 3-5 yêu cầu gửi lời chúc mỗi phút từ 1 địa chỉ IP).

## 🟢 Suggestions (Tùy chọn)

### 4. Thêm các Header bảo mật HTTP cơ bản
- **File:** [nginx/nginx.conf](file:///f:/DevOps/web/graduation_01/nginx/nginx.conf)
- **Lợi ích:** Tăng cường khả năng tự bảo vệ của trình duyệt khách truy cập khỏi các cuộc tấn công Clickjacking, Cross-Site Scripting (XSS) và MIME-sniffing.
- **Cách sửa:** Bổ sung cấu hình add_header trong block server của Nginx:
  ```nginx
  add_header X-Frame-Options "SAMEORIGIN";
  add_header X-Content-Type-Options "nosniff";
  add_header X-XSS-Protection "1; mode=block";
  ```
