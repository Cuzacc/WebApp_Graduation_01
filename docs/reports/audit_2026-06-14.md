Tài liệu này tóm tắt kết quả đánh giá bảo mật và các hoạt động dọn dẹp hệ thống trước khi triển khai ứng dụng lên môi trường sản xuất Google Cloud Platform (GCP).

## 🔒 Đánh Giá Bảo Mật & Các Cơ Chế Phòng Thủ

### 1. Chống Brute-force Đăng Nhập Admin (Rate Limiting)
*   **Triển khai:** Được cấu hình tại lớp Proxy Nginx (`nginx/nginx.conf`).
*   **Cơ chế:** Sử dụng `limit_req_zone` và `limit_req` với vùng nhớ `admin_login_limit`.
*   **Giới hạn:** Tối đa 5 yêu cầu/phút (`rate=5r/m`) từ cùng một địa chỉ IP, cho phép độ trễ ngắn (burst=3 nodelay). Khi vượt ngưỡng, Nginx sẽ trả về mã HTTP 429 (Too Many Requests).
*   **Đánh giá:** An toàn. Giúp bảo vệ backend hoàn toàn khỏi các cuộc tấn công dò quét mật khẩu tự động mà không gây quá tải cho tiến trình Python.

### 2. Quản Lý Phiên Làm Việc Quản Trị (Session Management)
*   **Triển khai:** Hàm đăng nhập quản trị trong `backend/main.py`.
*   **Cơ chế:** Khi đăng nhập thành công, hệ thống sinh ra một token ngẫu nhiên độ an toàn cao (`secrets.token_urlsafe(32)`) và lưu vào bảng `admin_sessions` kèm thời gian hết hạn (`expires_at` sau 24h).
*   **Xác thực:** Mỗi yêu cầu quản trị kiểm tra token động này, hạn chế rủi ro lộ khóa tĩnh `ADMIN_KEY` gốc.
*   **Đánh giá:** Đạt tiêu chuẩn bảo mật phiên làm việc cơ bản.

### 3. Chống Rò Rỉ Tài Nguyên (No Resource Leak)
*   **Triển khai:** API xóa vĩnh viễn `/api/admin/wishes/{id}/hard` và module `backend/utils.py`.
*   **Cơ chế:** Khi thực hiện xóa vĩnh viễn thiệp chúc mừng, hệ thống sẽ tự động duyệt toàn bộ các URL ảnh/video được liên kết (`wish.image_url` và mảng JSON `wish.image_urls`) để xóa vật lý:
    *   *Local:* Xóa tệp tin tương ứng trong thư mục `uploads/`.
    *   *GCP Storage:* Gọi Google Cloud Storage SDK để xóa tệp tin trực tiếp khỏi Bucket.
*   **Đánh giá:** Giải quyết triệt để vấn đề rò rỉ file rác làm tốn tài nguyên và phát sinh chi phí trên GCP.

### 4. An Toàn Cấu Hình (Configuration Security)
*   **CORS Policy:** Đọc động danh sách tên miền được phép từ biến môi trường `ALLOWED_ORIGINS` trong `.env`.
*   **Lộ lọt thông tin:** File `.env` chứa mật khẩu cơ sở dữ liệu và mã khóa Admin được cấu hình nằm trong `.gitignore` một cách chính xác, đảm bảo không bị đẩy lên Git Repository công cộng.
